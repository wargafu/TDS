# Mission de TDS

## Pourquoi TDS existe

Les services numériques gouvernementaux du Tchad souffrent d'une fragmentation visuelle :
chaque ministère, chaque préfecture, chaque organisme public développe ses interfaces
de manière indépendante, produisant des expériences incohérentes pour les citoyens.

Cette fragmentation a un coût réel :
- Les citoyens peinent à reconnaître les services officiels des imposteurs
- Les agents de développement répètent le même travail dans chaque équipe
- Les interfaces non accessibles excluent une partie de la population
- La maintenance est coûteuse car les solutions ne sont pas partagées

**TDS apporte une réponse systémique à ce problème.**

---

## Pour qui

### Les développeurs
Un point de départ unique pour toute interface d'État.
Plus de décisions à prendre sur les couleurs, les tailles de boutons, les espaces entre champs.

### Les équipes produit
Des composants pré-validés pour l'accessibilité, ce qui réduit les coûts d'audit.
La documentation claire permet aux designers de collaborer avec les développeurs sans ambiguïté.

### Les citoyens
Une identité cohérente pour tous les services de l'État, qui renforce la confiance
et réduit la charge cognitive lors de démarches administratives.

### L'État
Un investissement mutualisé — chaque amélioration de TDS bénéficie à toutes les équipes
qui l'ont adopté, simultanement.

---

## Principes de gouvernance

### Open source
TDS est publié en open source. Toute institution peut l'adopter, contribuer,
et adapter les tokens à son identité propre dans le respect des principes fondamentaux.

### Initiative ouverte au service du public
TDS est actuellement porté de manière indépendante par sa communauté de contributeurs.
Son code est ouvert, auditable et destiné aux institutions publiques tchadiennes.

### Évolution concertée
Les changements significatifs (nouveaux composants, modifications de tokens existants)
font l'objet d'une consultation ouverte avant implémentation.

## Inspirations

TDS s'inspire des Design Systems gouvernementaux les plus matures au monde :

| Système | Pays | Ce qu'on en a retenu |
|---|---|---|
| [GOV.UK Design System](https://design-system.service.gov.uk/) | Royaume-Uni | HTML-first, accessibilité non négociable, sobriété |
| [DSFR](https://www.systeme-de-design.gouv.fr/) | France | Tokens structurés, documentation complète |
| [USWDS](https://designsystem.digital.gov/) | États-Unis | Token system, multi-frameworks |
| [Inclusive Components](https://inclusive-components.design/) | — | Patterns accessibles profonds |
