# Versionnement

TDS suit le **versionnement sémantique** (semver) de manière stricte.
La stabilité des tokens et exports est une garantie envers les institutions utilisatrices.

---

## Règles semver

| Version | Format | Déclenché par |
|---|---|---|
| Patch | `0.1.X` | Corrections de bugs sans rupture, corrections typographiques |
| Minor | `0.X.0` | Nouveaux tokens, nouveaux composants (additifs) |
| **Major** | **`X.0.0`** | **Suppression d'export, modification de valeur de token, rupture d'API** |

---

## Garanties par version

### Tokens publiés

<Aside type="danger" title="Immuabilité des tokens">
  Une valeur de token publiée dans une version **ne change jamais** sans version majeure.
  `--tds-color-blue-500: #0033A0` restera `#0033A0` pour toute la durée de la version `0.x`.
</Aside>

### Exports npm

Les chemins d'export (`@tds-tchad/core`, `@tds-tchad/core/tokens`, `@tds-tchad/core/components/button`)
ne disparaissent jamais sans version majeure.

### Types TypeScript

Les types exportés (`ButtonVariant`, `InputSize`, etc.) sont stables.
Un nouveau champ optionnel peut être ajouté, mais jamais un champ requis sans version majeure.

---

## Dépréciation

Avant toute suppression, TDS suit ce processus :

1. **Minor** — Le token/export est marqué `@deprecated` dans les types TypeScript
2. **Minor suivant** — Un avertissement console est ajouté si possible
3. **Major** — Suppression effective

```typescript
/**
 * @deprecated Utiliser `color.blue[500]` à la place. Sera supprimé en v2.0.
 */
export const primaryBlue = '#0033A0';
```

---

## Cycle de releases

| Canal | Fréquence | Stabilité |
|---|---|---|
| `latest` (stable) | Mensuelle | Production |
| `next` (beta) | Hebdomadaire | Test, intégration |
| `canary` | À chaque commit | Développement uniquement |

```bash
# Installer la version stable
pnpm add @tds-tchad/core

# Installer la beta
pnpm add @tds-tchad/core@next

# Vérifier la version installée
pnpm list @tds-tchad/core
```

---

## Migration entre versions majeures

Un guide de migration sera publié pour chaque version majeure.
Il détaillera :
- Les tokens modifiés ou supprimés
- Les exports restructurés
- Les scripts de migration automatique si possible

---

## Politique de support

| Version | Support actif | Support sécurité |
|---|---|---|
| v0.x (actuelle) | Jusqu'à v1.0 | 6 mois après v1.0 |
| v1.x (future) | Jusqu'à v2.0 | 12 mois après v2.0 |

Les institutions sont encouragées à rester sur la version stable
et à migrer dans les 6 mois suivant une version majeure.
