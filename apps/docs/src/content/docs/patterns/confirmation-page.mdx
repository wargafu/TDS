# Pattern — Page de confirmation

La page de confirmation est affichée après la soumission réussie d'une demande.
C'est le moment de rassurer le citoyen, de lui confirmer les prochaines étapes
et de lui fournir une référence pour le suivi.

## Points critiques

- **Ne pas permettre la resoumission** — utiliser le pattern POST/redirect/GET
- **Toujours fournir un numéro de référence** — c'est la preuve de la transaction
- **Confirmer par e-mail** — doublon rassurant pour le citoyen
- **Indiquer les prochaines étapes** — le citoyen doit savoir ce qui se passe
- **Proposer un retour clair** — accès au tableau de bord ou à l'accueil
