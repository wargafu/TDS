# Patterns gouvernementaux

Les patterns sont des **solutions standardisées** à des problèmes d'interface récurrents.
Ils combinent les tokens et composants DSTD en structures prêtes à l'emploi.

---

## Pourquoi des patterns ?

Sans patterns partagés, chaque équipe réinvente :
- Le formulaire de connexion
- La page d'erreur
- La confirmation d'action

Le résultat : des interfaces incohérentes qui désorientent les citoyens
qui naviguent entre plusieurs services de l'État.

## Principes des patterns DSTD

1. **HTML-first** — chaque pattern fonctionne sans JavaScript
2. **Accessible par défaut** — WCAG AA respecté dans chaque pattern
3. **Mobile-first** — conçus pour fonctionner sur écran 320px
4. **Institutionnel** — sobre, sans effets superflus
5. **Complet** — chaque pattern inclut les états d'erreur
