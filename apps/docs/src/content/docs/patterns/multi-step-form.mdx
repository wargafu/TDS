# Pattern — Formulaire multi-étapes

Les démarches administratives complexes (demande de document, inscription, déclaration)
nécessitent souvent plusieurs étapes. Ce pattern guide les citoyens à travers le processus
sans les submerger.

---

## Principes

- Afficher une seule étape à la fois — ne pas tout montrer sur une page
- Indiquer la progression clairement (étape X sur Y)
- Permettre de revenir en arrière sans perdre les données
- Valider chaque étape avant de passer à la suivante
- Permettre de sauvegarder et reprendre plus tard (pour les formulaires longs)

## Points critiques

<Aside type="tip" title="Sauvegarde des données">
  Ne pas perdre les données saisies si l'utilisateur clique "Retour".
  Stocker les valeurs en session côté serveur ou en `sessionStorage` côté client.
</Aside>

- **Valider à chaque étape**, pas seulement à la fin
- **Le bouton "Retour" ne doit pas supprimer** les données de l'étape actuelle
- **Indiquer le nombre total d'étapes** dès la première étape
- **Permettre la navigation vers les étapes déjà complétées**
- **Afficher un résumé** avant la soumission finale (étape de confirmation)
- **Générer un token CSRF** sur chaque formulaire d'étape
