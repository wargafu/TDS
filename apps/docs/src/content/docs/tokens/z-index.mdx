# Z-index

La hiérarchie z-index DSTD est **verrouillée et non négociable**.
Les valeurs arbitraires hors de l'échelle sont interdites.

---

## Échelle

| Token | Variable CSS | Valeur | Usage |
|---|---|---|---|
| `base` | `--dstd-z-base` | `0` | Contenu de base, layout normal |
| `dropdown` | `--dstd-z-dropdown` | `100` | Menus déroulants, popovers |
| `sticky` | `--dstd-z-sticky` | `200` | Navigation fixe, barres collantes |
| `overlay` | `--dstd-z-overlay` | `300` | Backdrops masquants |
| `modal` | `--dstd-z-modal` | `400` | Fenêtres modales |
| `toast` | `--dstd-z-toast` | `500` | Notifications, toasts |
| `tooltip` | `--dstd-z-tooltip` | `600` | Tooltips, info-bulles |

**Ordre de priorité absolue :**
`tooltip (600)` > `toast (500)` > `modal (400)` > `overlay (300)` > `sticky (200)` > `dropdown (100)` > `base (0)`

---

## Règles strictes

- Les z-index arbitraires (ex: `z-index: 9999`) sont **interdits**
- `sticky` ne recouvre jamais une modale ou un overlay
- `overlay` bloque toujours l'interaction avec la page sous-jacente
- Ne pas créer de stacking contexts inutiles (`transform`, `opacity < 1`, `filter`, `will-change`)

<Aside type="caution" title="Stacking contexts">
  Les propriétés CSS `transform`, `opacity < 1`, `filter` et `will-change` créent
  un nouveau stacking context. Cela peut rendre un z-index élevé inopérant si l'élément
  parent a un contexte d'empilement avec une valeur plus basse.
  Vérifiez toujours la hiérarchie de contextes avant de déboguer les z-index.
</Aside>

---

## Variables CSS

```css
--dstd-z-base:     0;
--dstd-z-dropdown: 100;
--dstd-z-sticky:   200;
--dstd-z-overlay:  300;
--dstd-z-modal:    400;
--dstd-z-toast:    500;
--dstd-z-tooltip:  600;
```

```css
/* Exemple d'usage */
.ma-modale {
  z-index: var(--dstd-z-modal);    /* 400 */
  position: fixed;
}

.mon-backdrop {
  z-index: var(--dstd-z-overlay);  /* 300 */
  position: fixed;
  inset: 0;
}
```
