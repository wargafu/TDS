# Arrondis

Les arrondis DSTD sont volontairement limités. Un Design System d'État doit rester sobre :
les arrondis excessifs appartiennent aux interfaces startup, pas aux services publics.

---

## Échelle

| Token | Variable CSS | Valeur | Usage principal |
|---|---|---|---|
| `none` | `--dstd-radius-none` | `0` | Tables, sections formelles, containers de données |
| `xs` | `--dstd-radius-xs` | `2px` | Surfaces neutres, cards gouvernementales strictes |
| **`sm`** | **`--dstd-radius-sm`** | **`4px`** | **Boutons, inputs, tags — composants standard** |
| `md` | `--dstd-radius-md` | `8px` | Cards, modales, menus, sections de formulaire |
| `lg` | `--dstd-radius-lg` | `12px` | Dialogues, overlays, panels doux |
| `xl` | `--dstd-radius-xl` | `16px` | Sections majeures, encarts mis en avant |
| `full` | `--dstd-radius-full` | `9999px` | Badges, tags, pills, boutons full-round |

---

## Règles par composant

| Composant | Radius recommandé |
|---|---|
| Bouton | `sm` (4px) |
| Input, Textarea, Select | `sm` (4px) |
| Card standard | `xs` (2px) ou `md` (8px) selon la formalité |
| Modal, dialogue | `md` (8px) |
| Tableau de données | `none` ou `xs` |
| Badge, tag, pill | `full` (9999px) ou `sm` |
| Menu déroulant | `sm` (4px) |
| Notification, toast | `sm` (4px) |
| Tooltip | `xs` (2px) |
| Navigation / sidebar | `none` |

---

## Variables CSS

```css
--dstd-radius-none: 0;
--dstd-radius-xs:   2px   /* 0.125rem */
--dstd-radius-sm:   4px   /* 0.25rem  */
--dstd-radius-md:   8px   /* 0.5rem   */
--dstd-radius-lg:   12px  /* 0.75rem  */
--dstd-radius-xl:   16px  /* 1rem     */
--dstd-radius-full: 9999px
```

```css
/* Exemple d'usage */
.ma-card {
  border-radius: var(--dstd-radius-md);
}

.mon-badge {
  border-radius: var(--dstd-radius-full);
}
```
