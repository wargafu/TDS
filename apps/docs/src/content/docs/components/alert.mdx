# Alert

Les alertes communiquent des messages contextuels importants sans interrompre le flux utilisateur.
Elles existent en 4 variantes sémantiques alignées sur les couleurs du drapeau tchadien.

## Importation

```css
@import "@dstd/core/components/alert/alert.css";
```

```typescript
import type { AlertVariant } from '@dstd/core/components/alert';
```

|---|---|---|---|
| `info` | `dstd-alert--info` | `status` | Information neutre, aide contextuelle |

| `danger` | `dstd-alert--danger` | `alert` | Erreur bloquante, action critique |

## Accessibilité

<Aside type="tip" title="role alert vs status">
  - `role="alert"` + `aria-live="assertive"` — pour les erreurs critiques. Annoncé immédiatement.
  - `role="status"` + `aria-live="polite"` — pour les confirmations et infos. Annoncé au moment opportun.
  - N'utilisez `role="alert"` que pour les erreurs bloquantes — sinon les lecteurs d'écran interrompent constamment l'utilisateur.
</Aside>

```html
<!-- Erreur bloquante — annonce immédiate -->
<div class="dstd-alert dstd-alert--danger" role="alert" aria-live="assertive">
  ...
</div>

<!-- Confirmation — annonce polie -->
<div class="dstd-alert dstd-alert--success" role="status" aria-live="polite">
  ...
</div>
```
